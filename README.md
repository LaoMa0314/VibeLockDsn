# English ver.
With this skill, you can make your Vibe Design process rigorous and efficient, helping you realize agent-driven design, development, testing, and deployment for enterprise-grade products.
## How to Use This Skill:
- If you are a Chinese user, please use the two skills: VibeLockDsn-product-manager-zh and VibeLockDsn-product-developer-zh. Otherwise, if you are an English user, please use VibeLockDsn-product-manager and VibeLockDsn-product-developer.
- Please download and install these two skills into the skills root directory of your coding agent (such as Claude Code, Cursor, Trae, etc.).
- As a companion to these two skills, it is recommended to install Obsidian and pencil.dev software simultaneously. Meanwhile, you need to install Obsidian's official skill (primarily for controlling Obsidian via CLI and rendering Canvas images within Obsidian).
- Regarding styling, Impeccable is utilized here. Of course, this is not mandatory and depends on your UI styling preferences.
## What This Skill Can Do
- First, you can launch the first skill by using /VibeLockDsn-product-manager (-zh). It will guide you step-by-step to dissect development requirements, perform functional architecture planning, process architecture planning, data architecture planning, interaction planning, list feature lists, draw prototype UIs, as well as output PRDs and test cases.
- When using VibeLockDsn-product-manager (-zh), you can tell the agent your modification suggestions at each step to progressively achieve the desired effect.
- After completing all workflows of VibeLockDsn-product-manager (-zh), you can use /VibeLockDsn-product-developer (-zh) to perform vibe coding on the deliverables from the previous tasks, as well as conduct self-testing and packaging.
## How This Skill Differs from Other Usages
- Traditional vibe coding and vibe design often require guiding agents to complete design and development step by step through prompts and multi-turn conversations. For enterprise-level applications or rigorous projects, we often need to design complex data architectures and process architectures, which makes it extremely difficult to produce results solely through prompts. Therefore, with this methodology, I have incorporated the traditional workflow of enterprise-grade product management into the vibe design process, helping you realize agent-driven design, development, testing, and deployment for enterprise-grade products.
